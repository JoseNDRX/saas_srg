<p><strong>Technical PRD: Digital Identity SaaS Ecosystem</strong></p><p><strong>1. VISION &amp; CONTEXT</strong></p><p>The project is a dual-brand SaaS ecosystem designed to automate digital identity via micro-sites (vCards, Menus, and Showcases).</p><ul><li><strong>standmx.com</strong>: Commercial/Corporate entry point.</li><li><strong>puestoweb.com</strong>: Technical engine and constructor.</li><li><strong>MVP Goal</strong>: Create an ultra-fast mobile visualizer (PWA) that handles dynamic routing based on subscription tiers.</li></ul><p><strong>2. MODERN TECH STACK</strong></p><ul><li><strong>Frontend</strong>: React 18 (Vite) + Tailwind CSS + Framer Motion (for "App-like" transitions).</li><li><strong>Backend/BaaS</strong>: <strong>Supabase</strong> (PostgreSQL) for Auth, Database, and Storage.</li><li><strong>Routing/Edge</strong>: Vercel Edge Middleware for Wildcard DNS (*.standmx.com) and CNAME mapping.</li><li><strong>Infrastructure</strong>: SSL automation via Let's Encrypt or Vercel API.</li></ul><p><strong>3. SCHEMA DESIGN (SQL Server/PostgreSQL Compatible)</strong></p><p>SQL</p><p>-- Profiles &amp; Subscription Management</p><p>CREATE TABLE profiles (</p><p>  id UUID REFERENCES auth.users ON DELETE CASCADE PRIMARY KEY,</p><p>  full_name VARCHAR(255),</p><p>  tier VARCHAR(50) DEFAULT 'Free', -- [Free, Basic, Pro, Elite] </p><p>  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP</p><p>);</p><p>-- Microsites (The "Product")</p><p>CREATE TABLE microsites (</p><p>  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,</p><p>  owner_id UUID REFERENCES profiles(id),</p><p>  type VARCHAR(50), -- [vCard, Menu, Showcase] </p><p>  slug VARCHAR(100) UNIQUE, -- Used for puestoweb.com/[slug] </p><p>  subdomain VARCHAR(100) UNIQUE, -- Used for [client].standmx.com </p><p>  custom_domain VARCHAR(255) UNIQUE, -- For Elite CNAME </p><p>  config JSONB, -- Stores colors, logo URL, and layout [cite: 13, 19]</p><p>  content JSONB, -- Stores Menu items or vCard fields </p><p>  is_active BOOLEAN DEFAULT true</p><p>);</p><p>-- Smart QR &amp; Redirect Logic [cite: 14]</p><p>CREATE TABLE qr_analytics (</p><p>  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,</p><p>  microsite_id UUID REFERENCES microsites(id),</p><p>  short_code VARCHAR(10) UNIQUE, </p><p>  total_scans INTEGER DEFAULT 0</p><p>);</p><p><strong>4.
